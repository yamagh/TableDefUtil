### Java Model への制約アノテーション追加

`javax.validation.constraints` を利用して、入力値の検証を強化する。

#### 実装方針

1.  **NotNull 制約 (`@NotNull`)**
    *   `Constraint` カラムに `NN` が含まれているフィールドに対して、`@NotNull` アノテーションを付与する。

2.  **文字列長の制約 (`@Size`)**
    *   データ型が文字列型（`String`）のカラムを対象とする。
    *   `Constraint` カラムに `NN` が含まれている場合、`@Size(min=1, ...)` を設定し、空文字列を許容しないようにする。
    *   `Length` カラムに数値が指定されている場合、その値を `@Size(max=...)` として設定する。
    *   `NN` と `Length` の両方が指定されている場合は、`@Size(min=1, max=...)` のように両方の属性を設定する。

#### 具体例

**TSV定義:**

| 論理名 | 物理名 | データ型 | Length | Constraint |
| :--- | :--- | :--- | :--- | :--- |
| ユーザーID | userId | VARCHAR | 10 | NN |
| ユーザー名 | userName | VARCHAR | 50 | |
| メールアドレス | mailAddress| VARCHAR | 255 | NN |

**生成される Java Model:**

```java
import javax.validation.constraints.NotNull;
import javax.validation.constraints.Size;

public class User {

    @NotNull
    @Size(min = 1, max = 10)
    private String userId;

    @Size(max = 50)
    private String userName;

    @NotNull
    @Size(min = 1, max = 255)
    private String mailAddress;

    // getters and setters
}
```
