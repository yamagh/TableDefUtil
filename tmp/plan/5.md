## 5. Java Repository のページネーション対応

### 目的

Java の Repository クラス自動生成機能において、List を返す `find` および `findAll` メソッドにページネーション機能を追加する。これにより、大量のデータを一度に取得するのではなく、指定した件数や位置でデータを分割して取得できるようになる。

### 実装計画

1.  **`index.html` の Java Repository 生成ロジックの特定**:
    -   現在 `find` と `findAll` メソッドを生成している JavaScript コードの箇所を特定する。

2.  **メソッドシグネチャの変更**:
    -   `find` および `findAll` メソッドの引数に、オフセット（`offset`）と取得件数（`limit`）を追加する。
    -   例: `public List<Model> findAll(int offset, int limit)`

3.  **Ebean クエリの変更**:
    -   Ebean の `Query` オブジェクトに対して、`.setFirstRow(offset)` と `.setMaxRows(limit)` を呼び出すコードを追加する。
    -   これにより、発行される SQL に `OFFSET` 句と `LIMIT` 句が付与される。

4.  **UI への影響検討**:
    -   このページネーション機能を有効/無効にするための UI オプションを追加するか検討する。
    -   まずはデフォルトで有効として実装を進め、必要に応じて後からオプションを追加する方針とする。

5.  **生成コードの確認**:
    -   変更後に生成される Java Repository のコードが、構文的にも論理的にも正しいことを確認する。
